# 使用说明

流程：手机导出 VCF → 电脑整理 → 导出 VCF → 导入手机。

中间数据存放在 `data/contacts.sqlite`，通过 **DBeaver CE** 或 **LibreOffice Base** 编辑（二选一）。  
导出格式参照 `templates/format_sample.vcf`（vCard 4.0）。

姓名规范：`FN` 为全名；`N:;全名;;;`。导入时若 VCF 含分开的姓/名，程序会合并为全名。

项目概览与 FAQ 见 [README.md](README.md)；模块说明见 [docs/contacttools.md](docs/contacttools.md)。

---

## 环境准备

1. Python 3.10+
2. `pip install -r requirements.txt`（核心功能仅用标准库；`legacy/` 依赖 vobject）
3. 可视化工具（任选其一）：
   - [docs/dbeaver-setup.md](docs/dbeaver-setup.md)
   - [docs/base-setup.md](docs/base-setup.md)

写库脚本在修改 SQLite 前显示变更预览，确认后执行。DBeaver 直接改表不在此范围内。

**SQLite 版本**：视图语法需兼容 SQLite 3.40+（Debian 12 系统 Python）。若在较新系统（Debian 13）上建库后在旧系统跑脚本报错，请用最新代码执行一次 `init_schema` 重建视图。详见 [README.md — FAQ](README.md#faq)。

## 操作步骤

| 步骤 | 命令 | 说明 |
|------|------|------|
| 0 | `python 0-reset.py` | 可选，清空数据库 |
| 1 | `python 1-import_vcf.py` | VCF 写入 SQLite |
| 2 | `python 2-edit_in_dbeaver.py` 或 `2-edit_in_base.py` | 可视化编辑 |
| 3 | `python 3-dedup.py` | 去重与审阅 |
| 4 | `python 4-export_vcf.py` | 导出 VCF |
| 5 | `python 5-add_contact.py` | 从文本批量新增（可选） |
| 6 | `python 6-del_contact.py` | 按 id 删除（可选） |
| 7 | `python 7-merge_contact.py` | 按 id 合并（可选） |

步骤 2 两个脚本择一即可。步骤 5～7 可在任意阶段按需使用。

### 步骤 0：重置

```bash
python 0-reset.py
```

显示待删除记录数，确认后清空 `data/contacts.sqlite`。不删除 `input/` 内文件及 `legacy/`。

### 步骤 1：导入

1. 将手机导出的 VCF 置于 `input/contacts.vcf`
2. 按需修改 `1-import_vcf.py` 顶部：

```python
INPUT_VCF = "input/contacts.vcf"
IMPORT_MODE = "replace"   # replace | append | upsert
```

3. 执行 `python 1-import_vcf.py`

#### 导入模式

| 模式 | 适用 | 行为 |
|------|------|------|
| `replace` | 第一次整理 | 清空库后，VCF 全部写入 |
| `append` | 第二次及以后 | 完全一致跳过，有差异则新增；配合步骤 3 去重 |
| `upsert` | 以手机为准同步 | 同号合并更新，否则新增 |

**replace** — 清空库后全量导入。DBeaver 中尚未保存的修改会丢失。

**append** — 逐条与库中全部联系人比对内容（姓名、电话、邮箱、备注、单位等，不含 `id`）：

| 与库中某条的关系 | 结果 |
|------------------|------|
| 内容完全一致 | 跳过 |
| 任一字段不同 | 新增一条（不修改已有行） |

导入阶段不合并。同号两条、同名两条等在步骤 3 人工 `merge` 或删行。

典型第二次流程：

```
1-import_vcf.py  (append) → 可选 DBeaver 微调
3-dedup.py       → 【B】同名、【C】重复号码
4-export_vcf.py
```

**upsert** — 按号码判断：已在库中则合并进该条（保留原 `id`），否则新增。

| 字段 | 合并规则 |
|------|----------|
| `fn`、`org`、`note`、`url` | VCF 非空则采用，否则保留库中值 |
| 电话、邮箱 | 并集，按号码/地址去重，不删库中已有项 |
| `photo` | VCF 含头像则更新 |

#### 模式选择

| 场景 | 建议 |
|------|------|
| 第一次导入 | `replace` |
| 库为主、手机补新人；同名/重复靠步骤 3 | `append` |
| 手机为主、同号要同步回库 | `upsert` |

边界情形见 [README.md — FAQ](README.md#faq)。

### 步骤 2：编辑

**DBeaver** — `python 2-edit_in_dbeaver.py`，按终端输出新建 SQLite 连接。详见 [docs/dbeaver-setup.md](docs/dbeaver-setup.md)。

**LibreOffice Base** — `python 2-edit_in_base.py`。详见 [docs/base-setup.md](docs/base-setup.md)。

#### 推荐表/视图

| 名称 | 用途 |
|------|------|
| `v_contacts_wide` | 宽表浏览（tel1～tel5） |
| `v_contacts_full` | 完整浏览（全部号码/邮箱） |
| `phones` | 号码增删改 |
| `v_duplicate_phones` | 重复号码汇总 |
| `v_duplicate_phone_detail` | 重复号码明细 |
| `contacts` | 姓名、ORG、NOTE |

勿修改 `contacts.id`。保存并关闭编辑器后再执行步骤 3、4。

### 步骤 3：去重与审阅

```bash
python 3-dedup.py
```

`append` 导入后应执行。脚本按固定顺序列出问题，并在终端与 `data/review_warnings.txt` 中附带 SQL 与 DBeaver 操作说明。

| 步骤 | 内容 | 处理 |
|------|------|------|
| 1 | 完全相同的条目 | 删多余，每组保留一条；可自动删除 |
| 2 | 姓名相同、内容不同 | 同一人 → merge；不同人 → 改 `fn` |
| 3 | 电话号码相同 | 同一人 → merge；删错条 → DBeaver 删行 |
| 4 | 姓名格式、无号码等 | 按 SQL 提示修正 |

步骤 2、3 不会自动合并（步骤 1 完全重复可在确认后自动删除）。

```bash
python 7-merge_contact.py --keep <保留id> --drop <删除id> [<id> ...]
# 或
python -m contacttools merge --keep <保留id> --drop <删除id> [<id> ...]
```

#### DBeaver 快速参考

| 目的 | SQL |
|------|-----|
| 浏览全部 | `SELECT * FROM v_contacts_full ORDER BY fn_pinyin COLLATE NOCASE;` |
| 查重复号码 | `SELECT * FROM v_duplicate_phone_detail ORDER BY dup_count DESC;` |
| 按 id 查看 | `SELECT * FROM v_contacts_full WHERE id IN ('...');` |
| 删整人 | `DELETE FROM contacts WHERE id = '...';` |

### 步骤 4：导出

```bash
python 4-export_vcf.py
```

默认生成 `contacts_YYYYMMDD_HHMMSS.vcf`，传至手机后在联系人应用中导入。

### 步骤 5：批量新增

从文本文件新增联系人，无需 VCF。

1. 编辑 `input/add_contacts.txt`（或修改 `5-add_contact.py` 中 `INPUT_FILE`）
2. 每行：姓名 + 电话（必填），单位、备注（可空）；推荐 Tab 分隔
3. 同名多行合并为一人、多个号码
4. 执行 `python 5-add_contact.py`

```
张三	13800138000	某某公司	重要客户
张三	13900139000	某某公司
李四 059612345678
```

空行与 `#` 开头行忽略。写库前显示预览，可检测号码与库内已有联系人冲突。

### 步骤 6：删除联系人

```bash
python 6-del_contact.py <id1> [<id2> ...]
```

或在脚本顶部 `CONTACT_IDS` 中填写 `contacts.id`。删除 `contacts` 行后，`phones`/`emails` 级联删除。

### 步骤 7：合并联系人

```bash
python 7-merge_contact.py --keep <保留id> --drop <删除id1> [--drop <id2> ...]
python 7-merge_contact.py --keep <id> --drop <id> --mode simple   # 简单模式
```

- **interactive**（默认）：字段、号码逐项选择保留哪侧
- **simple**：keep 侧为主，空字段互补，电话/邮箱并集

## SQL 整理语句

以下语句均在 DBeaver 中实测可用，**保留于本文档**。  
同内容另整理为 `DBeaver/Scripts/` 下的 `.sql` 文件（写入脚本分预览/写入两段、带 ⚠️ 标注），见 [DBeaver/Scripts/README.md](DBeaver/Scripts/README.md)。两处可对照使用，**以本文档为准**。

在 DBeaver SQL 编辑器中执行。含 `UPDATE`/`DELETE` 的语句改前先 `SELECT` 预览，确认后再执行。

### 基础查询

```sql
-- 统计条数
SELECT COUNT(*) FROM contacts;

-- 姓名含空格（半角 U+0020 或全角 U+3000）
SELECT *, hex(fn) AS fn_hex FROM contacts
WHERE fn LIKE '% %' OR fn LIKE '%　%';

-- 姓名含特定字，列出其电话
SELECT p.id, p.contact_id, p.number, p.types, c.fn
FROM phones p
JOIN contacts c ON c.id = p.contact_id
WHERE c.fn LIKE '%工%'
ORDER BY p.id;

-- 596 开头改 0596（先预览）
SELECT p.id, c.fn, p.number, p.types
FROM phones p JOIN contacts c ON p.contact_id = c.id
WHERE p.number LIKE '596%'
ORDER BY p.number;

UPDATE phones SET number = '0' || number WHERE number LIKE '596%';
```

### 姓名结构

```sql
-- 姓氏非空，或名字为空
SELECT * FROM contacts
WHERE n_family IS NOT NULL
   OR n_given IS NULL
   OR TRIM(n_given) = '';

-- 姓/名长度异常
SELECT * FROM contacts
WHERE LENGTH(n_family) > 2 OR LENGTH(n_given) > 3;

-- 清空姓氏，令 fn = 名字（例：姓张名三 → fn=张三，n_family=NULL）
-- 预览
SELECT n_family || n_given AS 新名, * FROM contacts
WHERE n_family IS NOT NULL AND TRIM(n_family) <> ''
  AND n_given IS NOT NULL AND TRIM(n_given) <> '';

-- 执行
UPDATE contacts
SET fn = n_family || n_given,
    n_given = n_family || n_given,
    n_family = NULL
WHERE n_family IS NOT NULL AND TRIM(n_family) <> ''
  AND n_given IS NOT NULL AND TRIM(n_given) <> '';
```

### 电话结构

```sqlite
-- 仅查询
SELECT
    c.fn AS 姓名,
    p.id AS phone_id,
    p.number AS 电话,
    p.types AS 类型,
    p.pref,
    p.sort_order,
    CASE
        WHEN
            (LENGTH(p.number) = 11 AND p.number GLOB '1[3-9][0-9]*')
            OR (p.number GLOB '0*' AND LENGTH(p.number) BETWEEN 10 AND 12)
            OR (p.number GLOB '400*' AND LENGTH(p.number) = 10)
            OR (p.number GLOB '800*' AND LENGTH(p.number) = 10)
            OR (p.number GLOB '95*' AND LENGTH(p.number) BETWEEN 5 AND 8)
            OR (p.number GLOB '96*' AND LENGTH(p.number) BETWEEN 5 AND 8)
        THEN ''
        ELSE '异常'
    END AS 标记
FROM phones p
JOIN contacts c
    ON c.id = p.contact_id
WHERE p.contact_id IN (
    SELECT contact_id
    FROM phones
    WHERE NOT (
        (LENGTH(number) = 11 AND number GLOB '1[3-9][0-9]*')
        OR (number GLOB '0*' AND LENGTH(number) BETWEEN 10 AND 12)
        OR (number GLOB '400*' AND LENGTH(number) = 10)
        OR (number GLOB '800*' AND LENGTH(number) = 10)
        OR (number GLOB '95*' AND LENGTH(number) BETWEEN 5 AND 8)
        OR (number GLOB '96*' AND LENGTH(number) BETWEEN 5 AND 8)
    )
)
ORDER BY c.fn, p.sort_order, p.id;

-- 筛选
SELECT *
FROM phones
WHERE id IN (278,1054,668,1017,782,827,864)
ORDER BY id;

-- 电话为空的
SELECT
    p.id AS phone_id,
    c.fn AS 姓名,
    p.number AS 电话,
    p.types AS 类型,
    p.pref,
    p.sort_order
FROM phones p
LEFT JOIN contacts c
    ON c.id = p.contact_id
WHERE p.number IS NULL
   OR TRIM(p.number) = ''
ORDER BY c.fn, p.sort_order;
```

### 去重查询

内置视图已覆盖常见场景：

```sql
SELECT * FROM v_duplicate_phone_detail ORDER BY dup_count DESC, fn_pinyin, number;
SELECT * FROM v_duplicate_phones ORDER BY contact_count DESC;
```

**姓名重复**（自定义明细）：

```sql
SELECT p.id AS phone_id, c.id AS contact_id, c.fn, c.fn_pinyin,
       c.org, c.note, p.number, p.types, cnt.重复次数
FROM phones p
JOIN contacts c ON p.contact_id = c.id
JOIN (
    SELECT fn, COUNT(*) AS 重复次数
    FROM contacts WHERE TRIM(fn) <> ''
    GROUP BY fn HAVING COUNT(*) > 1
) cnt ON c.fn = cnt.fn
ORDER BY cnt.重复次数 DESC, c.fn, p.number;
```

**电话重复**（自定义明细）：

```sql
SELECT p.id AS phone_id, c.fn, c.fn_pinyin, p.number, p.types, cnt.重复次数
FROM contacts c
JOIN phones p ON c.id = p.contact_id
JOIN (
    SELECT number, COUNT(*) AS 重复次数
    FROM phones WHERE TRIM(number) <> ''
    GROUP BY number HAVING COUNT(*) > 1
) cnt ON p.number = cnt.number
ORDER BY cnt.重复次数 DESC, c.fn, p.number;
```

**按 phone.id 批量删除**（DBeaver 可用变量 `${ids}`）：

```sql
SELECT p.id, c.fn, p.number, p.types
FROM phones p JOIN contacts c ON p.contact_id = c.id
WHERE p.id IN (496, 501, 507)   -- 或 IN (${ids})
ORDER BY p.id;

DELETE FROM phones WHERE id IN (496, 501, 507);
```

### 删除联系人

```sql
-- 无号码的联系人
SELECT c.id, c.fn, c.fn_pinyin
FROM contacts c
LEFT JOIN phones p ON c.id = p.contact_id
WHERE p.contact_id IS NULL
ORDER BY c.fn;

-- 删除前预览
SELECT * FROM contacts WHERE id IN ('uuid-1', 'uuid-2');

-- 删整人（推荐）
DELETE FROM contacts WHERE id IN ('uuid-1', 'uuid-2');
```

## 常见问题

**找不到 VCF 文件**  
检查 `INPUT_VCF` 路径及 `input/contacts.vcf` 是否存在。

**Base 连接失败**  
确认 JDBC 驱动已加入类路径，连接 URL 为数据库绝对路径。

**DBeaver 能查、Python 报 schema 错误**  
系统 SQLite 版本低于建库时使用的版本。用最新代码重建视图，见 [README.md — FAQ](README.md#faq)。

**导入手机后重复**  
导入前备份；按手机提示选择跳过或覆盖重复项。

**旧版 CSV 流程**  
见 [legacy/README.md](legacy/README.md)。

其余（导入模式、database locked、导出内容不对等）见 [README.md — FAQ](README.md#faq)。
