# 常用整理语句

## 查询fn中带空格的

```sqlite
-- 查询联系人姓名(fn)及其十六进制编码，便于查看是否包含特殊字符
SELECT *, hex(fn) AS fn_hex
-- 从 contacts 联系人表中查询
FROM contacts
-- 筛选条件开始
WHERE fn LIKE '% %'      -- 姓名中包含普通空格（ASCII 空格，U+0020）
   OR fn LIKE '%　%';    -- 或姓名中包含全角空格（中文空格，U+3000）
```

## 查询fn中带"工"的

```
SELECT
    p.id,
    p.contact_id,
    p.number,
    p.types,
    (
        SELECT c.fn
        FROM contacts c
        WHERE c.id = p.contact_id
    ) AS fn
FROM phones p
WHERE p.contact_id IN (
    SELECT id
    FROM contacts
    WHERE fn LIKE '%工%'
)
ORDER BY p.id;
```

## 查询596开头，替换为0596

```sqlite
SELECT
    p.id,
    c.fn,
    p.number,
    p.types
FROM phones p
JOIN contacts c
    ON p.contact_id = c.id
WHERE p.number LIKE '596%'
ORDER BY p.number;

-- 执行修改
UPDATE phones
SET number = '0' || number
WHERE number LIKE '596%';
```

SELECT COUNT(*)

## 姓氏不为空或者名字为空/或超出多少字

```sqlite
SELECT *
FROM contacts
-- TRIM = 去掉字符串两端的空白字符。
WHERE n_family IS NOT NULL
   OR n_given IS NULL
   OR TRIM(n_given) = '';

SELECT *
FROM contacts
WHERE LENGTH(n_family) > 2
   OR LENGTH(n_given) > 3;
```

## 清空姓氏->让fn=名字

>姓：张 名：三 -> 姓：NULL 名：张三

```sqlite
-- 预览
SELECT n_family || n_given AS 新名, 
       *
FROM contacts
WHERE n_family IS NOT NULL
  AND TRIM(n_family) <> ''
  AND n_given IS NOT NULL
  AND TRIM(n_given) <> '';

-- 执行
UPDATE contacts
SET fn = n_family || n_given,
    n_given = n_family || n_given,
    n_family = NULL
WHERE n_family IS NOT NULL
  AND TRIM(n_family) <> ''
  AND n_given IS NOT NULL
  AND TRIM(n_given) <> ''
RETURNING *;
```

## 去重

### 姓名重复

```sqlite
SELECT
    p.id AS phone_id,
    c.id AS contact_id,
    c.fn,
    c.fn_pinyin,
    c.n_family,
    c.n_given,
    c.org,
    c.title,
    c.note,
    c.url,
    c.source,
    c.imported_at,
    c.updated_at,
    p.number,
    p.types,
    cnt.重复次数
FROM phones p
JOIN contacts c
    ON p.contact_id = c.id
JOIN (
    SELECT
        fn,
        COUNT(*) AS 重复次数
    FROM contacts
    WHERE TRIM(fn) <> ''
    GROUP BY fn
    HAVING COUNT(*) > 1
) cnt
    ON c.fn = cnt.fn
ORDER BY
    cnt.重复次数 DESC,
    c.fn,
    p.number,
    p.id;
```

编辑

```sqlite
SELECT
    p.*
FROM phones p
WHERE p.contact_id IN (
    SELECT id
    FROM contacts
    WHERE fn IN (
        SELECT fn
        FROM contacts
        WHERE TRIM(fn) <> ''
        GROUP BY fn
        HAVING COUNT(*) > 1
    )
)
ORDER BY p.contact_id, p.number;
```

### 电话重复

```sqlite
SELECT
    p.id AS phone_id,
    c.fn,
    c.fn_pinyin,
    p.number,
    p.types,
    cnt.重复次数
FROM contacts c
JOIN phones p
    ON c.id = p.contact_id

-- 统计每个重复号码出现了几次
JOIN (
    SELECT
        number,
        COUNT(*) AS 重复次数
    FROM phones
    WHERE TRIM(number) <> ''
    GROUP BY number
    HAVING COUNT(*) > 1
) cnt
    ON p.number = cnt.number

WHERE c.id IN (
    SELECT DISTINCT p2.contact_id
    FROM phones p2
    JOIN (
        SELECT number
        FROM phones
        WHERE TRIM(number) <> ''
        GROUP BY number
        HAVING COUNT(*) > 1
    ) d
        ON p2.number = d.number
)

ORDER BY
    cnt.重复次数 DESC,
    c.fn,
    p.number,
    p.id;
```

```sqlite
-- SELECT（我要什么）
-- FROM（从哪里来）
-- WHERE（满足什么条件）
-- 查询：所有“电话号码重复”的联系人
SELECT c.fn,
       -- 联系人姓名（来自 contacts 表）
       c.fn,
       -- 电话号码（来自 phones 表）
       p.number,
       -- 该电话号码总共出现了几次（例如 2 次、3 次）
       cnt.重复次数,
       -- 电话类型（如 CELL、HOME、WORK、FAX）
       p.types,
       -- 联系人 ID（用于关联 contacts 表）
       p.contact_id
-- 从 phones（电话号码表）开始查询
FROM phones p
-- 将 phones 表与 contacts 表连接
-- p.contact_id = contacts.id
-- 这样就能根据联系人 ID 找到联系人姓名
JOIN contacts c
ON p.contact_id = c.id

-- 再连接一个临时统计结果（子查询）
JOIN (
    -- 按电话号码统计出现次数
    SELECT number,
           -- COUNT(*) 统计每个号码出现了多少行
           COUNT(*) AS 重复次数
    -- 从 phones 表统计
    FROM phones
    -- 排除空号码（空字符串）
    WHERE TRIM(number) <> ''
    -- 相同号码分为一组
    GROUP BY number
    -- 只保留出现次数大于 1 的号码
    -- 即真正重复的号码
    HAVING COUNT(*) > 1
) cnt

-- 将统计结果与原 phones 表匹配
-- 例如：
-- phones.number = 13800138000
-- cnt.number    = 13800138000
-- 就把重复次数关联回来
ON p.number = cnt.number
-- 最后排序
ORDER BY
         -- 先按重复次数排序（大的排前面）
         cnt.重复次数 DESC,
         -- 重复次数一样，再按电话号码排序
         p.number,
         -- 同一个号码下，再按联系人姓名排序
         c.fn;
```

# 使用说明

流程：手机导出 VCF → 电脑整理 → 导出 VCF → 导入手机。

中间数据存放在 `data/contacts.sqlite`，通过 DBeaver CE 或 LibreOffice Base 编辑（二选一）。  
导出格式参照 `templates/format_sample.vcf`（vCard 4.0）。

姓名规范：`FN` 为全名；`N:;全名;;;`。导入时若 VCF 含分开的姓/名，程序会合并为全名。

项目目录与模块说明见 [README.md](README.md)、[docs/contacttools.md](docs/contacttools.md)。

---

## 环境准备

1. Python 3.10+
2. `pip install -r requirements.txt`（新版核心功能仅用标准库；`legacy/` 依赖 vobject）
3. 可视化工具（任选其一）：
   - [docs/dbeaver-setup.md](docs/dbeaver-setup.md)
   - [docs/base-setup.md](docs/base-setup.md)

写库脚本（import、merge、reset）在修改 SQLite 前显示变更预览，确认后执行。通过 DBeaver 直接改表不在此范围内，修改后需自行核对。

## 步骤 1：导入

1. 将手机导出的 VCF 置于 `input/contacts.vcf`
2. 按需修改 `1-import_vcf.py` 顶部配置：

```python
INPUT_VCF = "input/contacts.vcf"
IMPORT_MODE = "replace"
```

3. 执行：

```bash
python 1-import_vcf.py
```

### 导入模式

| 模式 | 适用 | 行为 |
|------|------|------|
| `replace` | 第一次整理 | 清空库后，VCF 全部写入 |
| `append` | 第二次及以后 | 见下表；配合 `3-dedup.py` 人工去重 |
| `upsert` | 以手机为准同步 | 同号合并更新，否则新增 |

#### replace

清空 `contacts.sqlite` 后导入 VCF 全部联系人。DBeaver 中尚未保存的修改会一并丢失。

#### append

逐条与库中**全部**联系人比对内容（`fn`、电话及类型、邮箱、备注、单位、URL、头像等，不含 `id`）：

| 与库中某条的关系 | 结果 |
|------------------|------|
| 内容完全一致 | 跳过 |
| 任一字段不同 | **新增一条**（不修改已有行） |

示例：

- 库：张三，`138`、`139` → VCF 也是张三 `138`、`139` → **跳过**
- 库：张三，`138`、`139` → VCF 张三 `138`、`139`、**`140`** → **新增**（多第三个号）
- 库：**错名** + `138` → VCF：**正确名** + `138` → **新增**（姓名不同）
- 库：张三 + `138` → VCF 全新号码 `159` → **新增**

导入阶段**不合并**。同号两条、同名两条等，在步骤 3 由人工 `merge` 或删行。

典型第二次流程：

```
1-import_vcf.py  (IMPORT_MODE = "append")
    → 可选：DBeaver 微调
3-dedup.py       → 【B】同名、【C】重复号码，人工决定保留或 merge
4-export_vcf.py
```

```bash
python -m contacttools merge --keep <保留的id> --drop <要删的id> [<id> ...]
```

要在导入时**直接合并进已有行**（不新增 duplicate），用 `upsert`，见下。

#### upsert

逐条按号码判断：

| VCF 中的号码 | 结果 |
|--------------|------|
| 已在库中 | 合并进该条（保留原 `id`） |
| 不在库中 | 新增 |

同号不会在库中产生两条记录；字段以 VCF 与库中数据合并为准（见下）。

**upsert 合并规则**（`contacttools/ops/contact_merge.py`）：

| 字段 | 规则 |
|------|------|
| `fn`、`org`、`note`、`url` | VCF 非空则采用，否则保留库中值 |
| 电话、邮箱 | 合并为并集，按号码/地址去重，不删除库中已有项 |
| `photo` | VCF 含头像则更新 |

适合需要把手机里改过的姓名、备注**直接写回已有记录**、且不想在库中暂时出现两条的场景。若可接受「先新增、步骤 3 再 merge」，用 **append** 即可。

#### 模式选择

| 场景 | 建议模式 |
|------|----------|
| 第一次导入 | `replace` |
| 库为主、手机补新人；同名/重复靠步骤 3 处理 | `append` |
| 手机为主、同号联系人要同步回库 | `upsert` |

## 步骤 3：去重与审阅

```bash
python 3-dedup.py
```

`append` 导入后应执行本步骤。脚本按固定顺序列出问题，并在终端与 `data/review_warnings.txt` 中附带 **SQL 与 DBeaver 操作说明**。

| 步骤 | 内容 | 处理方式 |
|------|------|----------|
| 1 | 完全相同的条目 | 删多余，每组保留一条；脚本可询问后自动删除 |
| 2 | 姓名相同、内容不同 | 同一人 → `merge`；不同人 → 改 `fn` 区分 |
| 3 | 电话号码相同 | 同一人 → `merge`；删错条 → DBeaver 删行 |
| 4 | 姓名格式、无号码等 | 在 DBeaver 按 SQL 提示修正 |

**不会自动合并**步骤 2、3（步骤 1 完全重复可在确认后自动删除）。

```bash
python -m contacttools merge --keep <保留id> --drop <删除id> [<id> ...]
```

### DBeaver 快速参考

| 目的 | SQL |
|------|-----|
| 浏览全部 | `SELECT * FROM v_contacts_full ORDER BY fn_pinyin COLLATE NOCASE;` |
| 查重复号码 | `SELECT * FROM v_duplicate_phone_detail ORDER BY dup_count DESC;` |
| 按 id 查看 | `SELECT * FROM v_contacts_full WHERE id IN ('...');` |
| 删整人 | contacts 表选中行删除，或 `DELETE FROM contacts WHERE id = '...';` |

详细 SQL 见报告文件 `data/review_warnings.txt`。

## 常见问题

连接、路径、手机导入等见 [README.md](README.md#faq) 的 FAQ。此处补充操作层问题。

**找不到 VCF 文件**  
检查 `INPUT_VCF` 路径及 `input/contacts.vcf` 是否存在。

**Base 连接失败**  
确认 JDBC 驱动已加入类路径，连接 URL 为数据库绝对路径。

**导入手机后重复**  
导入前备份；按手机提示选择跳过或覆盖重复项。

**旧版 CSV 流程**  
见 [legacy/README.md](legacy/README.md)。
